Introduction

I was asked to lead the design effort to redo Cogo's offer managment tooling, Offer Coffer. Offer Coffer is an old, fragile tool that the organization relies on for revenue but was not designed or built to scale the way it is currently used. Because analytic and account managment teams rely on it every day, I needed to make sure that all of their needs were absolutely accomodated while also improving the user experience and reducing cognitive load.

Research

Because this tool is replacing an existing one, I had a lot of data on the current version to collect. This project was much more massive than anything that I or other designers in recent past had done before, and I needed a way to organize and synthesize the abundance of data I was about to collect.

To begin the process of remaking our offer management tool Offer Coffer, we conducted 15 user interviews between 7 teams. The goal was to capture main use cases, key needs, painpoints, and where we can create more solutions within the UI in the next version.

I then created a Zenhub board to document all qualitative data points collected from these interviews, organize them, and see emerging patterns to be used to inform the remake to best serve our userâ€™s needs.

Each issue has labels, creating an easy filtering system to isolate specific data points and patterns within a category (for example, seeing all data points collected on a specific feature).

Key Findings Report

After all of that data was collected on the interview board, I used the labels to filter by feature/category to find patterns within specific sections of the application. Processing this, I documented my key findings of these patterns and potential features that would help alleviate revealed pain points.

Going through this process not only helped me digest and analyze the data, but served as a source of truth during development conversations between myself, engineers, and stakeholders.
