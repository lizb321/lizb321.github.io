Introduction

I was asked to design a job searching site with the goal of making multiple copies for different brands. This was kept in mind throughout the entire process of designing the site, as to make it the most usable and specific to the brand while still keeping it replicable. You can view the site here.

I worked closely with the software engineer on the project to develop the site. They coded the back end while I coded the front end. I worked in Nuxt.js and Vue.js for the first time, using my HTML and CSS skills I already had.

Wireframes

The goal of Hire Harbor was to have a simple jobs site where a user could search for a job, view results, and sign up for alerts. These were all expressed in the wireframe. Going into the project it was known that there would be multiple reskins of the site, such as Hire Harbor, so I wanted to keep the design simple enough where text and styling could be swapped easily without much tweaking as the brands changed.

Style Guide

Homepage

Again, because I knew the site was to be reskinned for multiple brands, I wanted to create a layout that created the most efficient process possible. The goal was to have me or the next designer able to easily swap styling and images in the mockup and code without needing to make major tweaks for each new brand.

Job Alerts Modal

When a user clicks the "Get Alerts" button, a modal appears with a short description of telling them about alerts and a form for them to sign up.

Job Listings Page

The user is presented with job listings (from the Indeed.com API). Each listing contains the title, recency, company, location and brief description.

About Page

I included a short description of the site, and added a job alert button.

Marketing Emails

Before we have information on users' interests, we present them with a more generic CTA of choosing topics they're interested in. When we know information about a user, we can send them more specific marketing content about job listings with keywords and locations we know they're interested in.
